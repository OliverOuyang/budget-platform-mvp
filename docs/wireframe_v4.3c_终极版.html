📐 计算逻辑手册 V4.3c: 独立页面
本页用途：完整说明预算推算的计算逻辑、公式、参数来源和目标方案的影响方式。适合向领导汇报、新人理解系统、以及未来优化计算逻辑时的参考文档。
1总体流程
📂 上传Excel→🔍 质检+护栏检测→📐 提取历史系数→⚙️ 设定目标+参数→🔢 计算引擎→📊 结果展示
数据来源：raw_达成情况 (渠道维度月度数据) + raw_客群首借金额 (客群维度)
系数提取：自动从历史数据计算 M0/T0比值 和 存量M0 CPS
双引擎：V01(规则引擎，历史系数外推) + MMM(统计模型，Adstock+Hill回归)
2V01 核心公式 — Table 1 (渠道维度)
渠道花费 = 总花费 x 渠道占比
T0交易额(亿) = 渠道花费(万) / CPS / 10000
T0申完量 = 渠道花费(元) / T0申完成本(元)
当月首登M0交易额 = T0交易额 x M0/T0系数
1-3授信量 = T0申完量 x 1-3过件率
免费渠道特殊处理：CPS=0(无投放成本)，T0交易额使用历史数据外推而非上述公式计算。
3客群汇总 — Table 2
初审M0交易额 = 当月首登M0(各渠道之和) + 存量首登M0
初审合计 = 初审M0 + 初审M1+
整体首借交易额 = 初审合计 + 非初审交易额
全业务CPS = (投放花费 + RTA费用) / 整体首借交易额
存量M0交易额和初审M1+使用历史均值(3/6月)外推，非初审交易额为用户手动输入。
4关键系数来源
系数计算方式当前值稳定性说明
M0/T0比值最近6个月月度(M0交易额/T0交易额)的均值1.49CV=1.5%M0范围大于T0，系数>1
存量M0 CPS最近3个月月度CPS均值34.4%下降趋势用于计算存量M0交易额
渠道CPS用户输入 / MMM推荐各渠道不同—核心可调参数
过件率用户输入 / 历史均值加权14.6%—影响授信量计算
5目标方案如何影响计算
选择不同目标方向，系统自动调整不同参数，生成3个梯度(保守5%/标准10%/激进15%)的方案。
💰 降成本
CPS_new = CPS_old x (1 - 幅度)
例: 腾讯CPS 32.5% x (1-10%) = 29.25%
调整: CPS → CPS ↓ → T0交易额 ↑ → M0 ↑ → 首借 ↑ → 全业务CPS ↓
原理: CPS是T0交易额公式的分母，CPS降低直接导致T0增加，连锁影响M0和首借。
📈 提规模
总花费_new = 总花费_old x (1 + 幅度)
例: 3,000万 x (1+10%) = 3,300万
调整: 总预算 → 总花费 ↑ → 各渠道等比 ↑ → T0 ↑ → M0 ↑ → CPS 不变
原理: 预算等比放大，CPS不变因此效率不变，但绝对量增加。
⭐ 提质量
过件率_new = 过件率_old x (1 + 幅度)
例: 14% x (1+10%) = 15.4%
调整: 过件率 → 过件率 ↑ → 授信量 ↑ → 结构优化 → T0交易额 不直接变化
原理: 过件率影响授信量(1-3授信 = 申完量 x 过件率)，不直接改变T0交易额公式。
6MMM 方法论简述
模型结构
y = intercept + trend + ∑ coeff_i x Hill(Adstock(spend_i))
Adstock变换：Weibull PDF衰减，模拟广告滞后效应
Hill饱和变换：S曲线，模拟边际效益递减
回归方法：Ridge回归，非负系数约束
超参优化：Optuna TPE，300轮试验
V01 vs MMM 方法论差异
V01 规则引擎MMM 统计模型
方法历史系数外推Adstock+Hill回归
数据月度渠道数据周度花费时序
优势直观可解释捕捉饱和+滞后
局限假设线性关系依赖数据量
适用稳态预算估算渠道优化分配
